== What is syntax?

Rules and principles that govern the structure of sentences

* defines rules for combining words
** phrases
** clauses
** sentences

Syntax must define 3 properties:

. linear order
* the correct order in which to place these words
* order is significant in many languages
** the mouse eats the cat != the cat eats the mouse
. hierarchical structure
* groups of words are taken together to express a concept
** Bob's aunt is named Alice -> Bob's aunt is a hierarchical unit
* need to reconstruct *constituents*
. grammatical relations
* roles that each word/group of words/group of groups of words cover in a grammatical structure
** and therefore its semantic meaning

=== Chomsky hierarchy

Classification of formal languages

* less/more formal languages

3 types that fit into each other (0 contains 1 contains 2 contains 3)

. Type 3 (regular, parsed by finite state automata)
* A -> aB (non terminal produces nonterminal and terminal)
* A -> a (non terminal produces terminal and stops)

. Type 2 (context free grammars, parsed by push down automata)
* single nonterminal produces sequence of terminals and non terminals
* typically used in programming languages
** except Python apparently?

. Type 1 (context sensitive, linear bounded automata)
* can use thing only if near other thing
* context is not modified
** context doesn't change during production
** only if A is between alpha and beta can I produce gamma between alpha and beta

. Type 0 (recursively enumerable)
* meaning that strings produced by this language can be compared to integers???
** i.e. give me the next thing -> apply next production recursively
* can do anything, no restrictions
* futuristic poetry and things like medical notes don't always follow typical grammar rules

Natural languages are somewhere between type 1 and 2

* we can say/write nonsense
* context isn't always required/lack of context isn't always a big deal
** mildly context sensitive

===== Type 3 languages

Single non terminal -> terminal or terminal+non terminal on right

If in state A and I read symbol a I go to state B

* if in state A I generate a and go to state B
* or if in state A I generate a and stop

This mechanism has no memory

* only knows the current state

They cannot recognize languages with the same number of objects

They also cannot parse nested structures

* the patient was seen by the doctor who was in the ICU survived
** recursion on patient

Useful for pattern matching

* fiscal codes
* SSN
* regex

===== Type 2 languages

Adds a memory to type 3

* stack memory

teminal A -> sequence of terminals/non terminals gamma regardless of context

* classically used for programming languages

Can handle recursion but not parallel structures

* the nurse and physician administered the IV and performed the intubation respectively
* this string causes failure

Also no context sensitivity

* any noun is valid
* the him wrote an fish

Still useful for extracting information from text

* information extraction instead of language analysis/generation

===== Type 1 languages

Context changes

* information within the context can change
* uses special characters to denote context

===== Type 0 languages

No constraints

* all productions are possible
* anything that is computable can be expressed by type 0 languages
** parsing type 0 languages is impossible

=== Phase Structure Grammar (PSG)

Replicates traditional grammar

* transforms into computationally parsable stuff

Introduces lots of *non terminals*

* classes of *clauses*
** parts of a sentence

==== PSG Classes

. S - sentence
* represents entire grammatically complete unit
* arbitrary
** not necessarily topmost level of hierarchy

. NP - noun phrases
* group of words centered around a noun
* ways to denote a single entity (a dog, the elderly patient)

. VP - verb phrase
* group of words centered around a verb
** also contains adverbs, nouns, etc

. PP - prepositional phrase
* a phrase that uses a preposition to provide context
** the cat is *in* the closet

. ADJP - adjective phrases
* same but adjective

. ADVP - adverb phrase
* same but adverb
* multiple words can count as an adverb

. CP - complementizer phrase
* used for nested clauses
** I know that you know that he knows

. QP - quantifier phrase
* numeric/quantity based expressions

==== PSG productions

. S -> NP VP
* sentence -> noun phrase + verb phrase

. NP -> (det) (ADJP) N (PP)
* parentheses are optional
* like the green dog is in the closet

==== PSG Output

Produces *deep parse trees*

* concrete syntax tree
** represents structure
* called deep but can be shallow
** trees can closely resemble the syntactic surface of a sentence
* intermediate nodes represent the constituent clauses

=== Tree Adjoining Grammar (TAG)

Does not fall neatly within Chomsky's hierarchy

* inbetween 2 categories

Works directly on trees

* does not create trees from sequences
* trees can be manipulated

Defines 2 operations:

. adjunction
* given a tree alpha with an internal node X and a new tree beta with node X and internal node X
** take whole subtree and insert it between root tree and subtree
** joined at internal node X and new node X
. substitution
* all nodes are labeled (T/NT)
** terminals can be replaced with other non terminals (what)

What are alpha and beta?

* alpha trees are the grammar structures of the language
* beta trees are utilities
** essentially recusrion helpers

This separates production from recursion

* preserves locality
** the relationship between the targets of the phrase
** patient and operation are in the same alpha trees
** even when beta trees are inserted

TAG can parse all context independent languages

* some context dependent languages as well
* falls between Type 1 and 2

However it's very complex

* stem:[O(n^6)] (lmao)
** some versions can get down to stem:[O(n^3)] but you lose most of their power so what's the point

=== Dependency Grammar

Most commonly used outside LLMs

No phrasal nodes

No non terminals

* key part of Chomsky's structure
* fuck you Chomsky

Only terminals exist

* words, lemmas, morphemes, tokens

Focuses on relationships between these terminals

* one word is the head and the other is the dependent
** relationship is a *dependency
** one head can have many dependents
* edges labeled with the nature of the relationship between the nodes

So this is still a tree

* kind of
* typically not drawn like a standard tree
** nodes are all at the same hierarchy in the order in which they appear in the sentence
** edges arc over them

== Dependency Parsing

2 main ways to parse dependencies

=== Transition based methods

Constructs a tree through a sequence of local actions

Very efficient

* only have to look locally before deciding which edges to draw
* processes tokens linearly onto a stack

==== Transition Mechanism

Special kind of automaton

* *shift reduce* automaton

3 parts:

. parser (oracle)
* parses stuff
. stack
* holds currently processed word(s)
. input buffer
* words in the input

The oracle looks at:

* top 2 words in stack
* first word in buffer

Oracle can:

* manipulate the stack
* move through the input

Very efficient

* stem:[O(n)]
** large corpora are processed in reasonable time
** assuming the oracle is reasonably efficient as well

==== Parsing algorithm (shift +  reduce)

3 parts:

. input word
. top stack
. second top stack

3 actions:

. shift
* take current word in buffer and put into stack
* point or consume

. left arc
* creates dependency of given class between top 2 stack items and removes the dependent
* head <- dependency

. right arc
* same but reversed

=== Graph-based methods

Takes a graph

Builds a fully connected graph

* dense graph
** complexity at least stem:[O(n^2)]
* all nodes are connected
** every word is a potential head for every other word
* edges are then labeled and weighted
** weights themselves are not that important
** differences between weights are more important
* take subgraph with whatever properties we like
** e.g. *Maximum Spanning Tree*
** need each node to have exactly 1 incoming edge (except the root)

==== Finding MST

Using a greedy algorithm is generally a good idea for this task

* for each node find select incoming edge with max weight
** might have cycles -> not a tree

Using Chu-Liu, Edmonds algorithm

* recursive
* collapses all nodes in a cycle into single node
* apply algorithm recursively into collapsed node
** expand resulting node (should be a tree at this point)
** resulting nodes might be single nodes

Can have complexity between O(n^2) and O(n^3)

==== Assigning weights

Run feature engineering to get linguistic features

Weights are learned using ML on annotated corpora
